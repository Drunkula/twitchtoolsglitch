A player's playlist will appear here when you select one from the dropdown on the right.
Shift clicking works on checkboxes.
Double clicking an song will make the player play that song.
If you rearrange or delete from a list use the 'update list' button to update it in the player.
Disabling the auto option means you have to use the reload button to loads the player's list.  This can be used to 'get' a player's list, rearrange and delete from it, select a different player and send the list to it.
NOTE: send replaces the players entire list.

If you chose the 'All Players' option it'll be send everywhere, so be careful if you've got multiple players in your setup for things like start and end music. The changes are not permanent.

The add video box will add to the player targetted by the select or all players. Double click or right click the box to clear it.

Deleting videos from the player won't automatically update the list, you need to refresh to see the result. This lets you copy the entries to a playlist after if needed.

Playlist.

players now can have (and need) optional paramers. Most default to off

For a chat controlled list you'd want nan, chatadd, chatcmds to be on

https://twitch-tools.glitch.me/ytplayer/?id=Name in Observer Page&nan=true&chatadd=true&chatcmds=true

Put a ? after the url and separate them with & signs. E.g. /?playlist=default&x=1280&y=720

The options are:

  • playlist - default = default. Set with playlist=Start Music
  • id - the name that'll show in the controller page
  • nan - default: false. Set true for the player to respond to now and next requests and !added videos.
  • chatadd - default: false. true means it'll add videos from chat with !add or !src
  • chatcmds - default: false. true allows !next !prev !play !pause, etc from chat unless locked out
  • shuffle - true/false Override the shuffle flag for a list. Not included uses the list's default.
  • video. The id of the initial video to load e.g. video=xayd1c0wtOE
  • x and y to set the size of the player
  • muted - true/false. Whether the player starts muted. More for debugging
  • volume - 0 to 100. Initial volume of player. Default 100.
  • debug - true/false. Whether debug messages appear in a browser console.
  • NOTE: if you delete items from a playlist, don't refresh then rearrange and update the list the deleted entries will be resaved.
  • The yt_playlist_default length value in the variables table shows you the data length as the variable itself looks blank.

A playlists chat add option means it'll store chat !adds unless the permanent storage option is set to off. Having a list with this set means you can copy ones you like to other playlists later.

OBS
  • Player sources must start YTPlayer in OBS for them to be automatically paused/resumed on scene changes.
  • Players must be in the root of a scene for them to automatically pause/resume OR...
  • Player proxies can be used for players that are in groups or nested scenes or even not in the scene. e.g. for a hidden player used for background music.
  • Proxies are any sources called the player's source name with Proxy after and be in the scene's root. e.g
  • YTPlayerStartMusicProxy for YTPlayerStartMusic. No space. YTPlayer CartoonsProxy for YTPlayer Cartoons. Any source type can be used for a proxy.
  • The visibility of a player doesn't affect playing/resuming. If you've got a player hidden in a scene and don't want it to play then put it in a group.
